Краткое руководство по разрабртке Telegram бота.

Создание учетной записи бота и получение ключа

  1. Откройте Telegram и найдите бота BotFather. У настоящего бота рядом с именем будет иконка с галочкой.
  2. Введите команду /newbot. Так же вы можете ввести команду /start и выбрать команду /newbot из списка.
  3. Введите имя бота, например tetris (популярная в прошлом компьютерная игра)
  4. Далее введите имя пользователя для вашего бота. Оно должно оканчиваться на bot. Для бота tetris имя пользователя может быть TetrisBot или tetris_bot.
  5. Храните в секрете токен для доступа к HTTP API, который выдаст вам BotFather. Примерный токен: 5495174487:AAF_0XCVKmReqglXndevO3xzzE8ptK4SHJx

Создание проекта в PyCharm для работы с ботом

  1. Создайте проект в PyCharm. Для этого пройдите в меню File | NewProject. Откроется окно Create Project.
  2. Введите имя проекта и его расположение в поле Location окна Create Project.

Важно! Путь к проекту должен быть без кириллических символов.
Правильно: D:\Python\tetris_bot_project
НЕТ: D:\Мои Проекты\tetris_bot_project

  1. Нажмите кнопку Create. Нажмите This Window если PyCharm спросит в каком окне отображать новый проект.
  2. Установите библиотеку AIOGRAM. Для этого во вкладке Terminal введите команду
pip install aiogram
  • Версия AIOGRAM на момент написания гайда - 2.25.1 (aiogram-2.25.1)
  1. Подключите из AOIGRAM базовые классы для начала работы:
from aiogram import Bot, Dispatcher, executor, types

Готово. Приступайте к работе.

Базовый шаблон приложения (бота)

  1. Создаём экземпляр бота и привязываем к нему токен.
  2. Создаём экземпляр диспетчера для созданного бота.

Dispatcher обрабатывает входящие обновления: сообщения, отредактированные сообщения, записи в канале, отредактированные записи в канале, встроенные запросы, callback-запросы и т.д.

  1. С помощью executor`а запускаем опрос сервера ТГ.
bot = Bot(token=TOKEN)
dp = Dispatcher(bot=bot)

# ... тут описываются обработчики сообщений

if __name__ == '__main__':
    # start_polling - запускает опрос сервера ТГ
    # skip_updates=True позволяет пропустить накопившиеся входящие сообщения
    executor.start_polling(dp, skip_updates=True)

Обработка сообщений.

Бот может реагировать на специальные сообщения (команды ) и обычные сообщения. Сообщения могут быть отправлены напрямую боту или в чат группы, где есть бот.
Обработка сообщений осуществляется с помощью декоратора @dp.message_handler(). Этот декоратор позволяет достать из диспетчера нужное сообщение и “приклеить” к нему функцию-обработчик.

Обработка команд

Команды - специальные сообщения, которые начинаются с символла /. Команды нужны для активации какой-либо функции бота.

  1. Для обработки одной команды используйте декоратор @dp.message_handler() с параметром commands, в который передайте строку с командой - commands='start'.

Параметр commands может принимать несколько команд, на которые будет реагировать функция-обработчик. Для этого передайте параметру список команд:
commands=['start', 'go'].

  1. Объявите функцию-обработчик команды ниже декоратора. В этой функции напишите “реакцию” бота на команду.

Функция-обработчик - асинхронная, поэтому перед def используется оператор async (он и делает функцию асинхронной).
Результат работы асинхронной функции-обработчика будет передаваться другим асинхронным функциям (или методам). Поэтому вместо return используется await.

@dp.message_handler(commands='start')
async def start_handler(message: types.Message): # type hint
    user_name = message.from_user.full_name
    await message.reply(f'Привет, {user_name}!')

Для обработки “сообщений”, которые получет бот, функции-обработчику нужно передать объект типа Message (т.е. объект сообщения). Делается это с помощью явного указания типа параметра (type hinting`а) для функции-обраотчика- start_handler(message: types.Message).

Код выше позволяет боту отреагировать на команду /start: в ответ на неё бот поздоровается с пользователем по имени, которое он “узнает” из сообщения (объекта Message).
Текст приветствия (в виде цитаты) пользователю отправляет асинхронная функция message.reply().

Сообщение от пользователя - не просто текст. Оно несёт гораздо больше информации.
Лао Цзы

Объект Message

Чтобы получить доступ к сообщению от пользователя, необходимо в функцию-обработчик передать параметр типа Message. Этот объект позволяет получить не только текст сообщения, но и другую важную информацию.

Информация о пользователе (атрибут from_user):
message.from_user.id - Уникальный идентификатор пользователя вида 239030841
message.from_user.first_name - Имя пользователя, например, Игорь
message.from_user.last_name - Фамилия пользователя, например, Иванов
message.from_user.username - То, что через @, например @ivanov_igor
message.from_user.full_name - Фамилия и имя вместе: Иванов Игорь

Информация о дате сообщения (атрибут date):
message.date - полная дата и время 2023-04-06 17:24:25
message.date.hour - а так можно получить только часы из даты (17)
message.date.minute - а так - минуты (24)
message.date.day - получаем только день (6)

Обработка обычных сообщений

Научить реагировать бота на обычные сообщения можно с помощью lambda-функции и фильтра Text. Эти способы позволяют определить конкретную фразу, на которую отреагирует бот.

Чтобы написать обработчик сообщений с помощью lambda-функции, передайте её декоратору @dp.message_handler():

@dp.message_handler(lambda message: message.text == 'Привет, друг!')
async def second_button(message: types.Message):
    await message.answer('И тебе привет, Леонид!')

Внимательно! Чтобы Бот отреагировал на сообщение, оно должно в точности соответствовать строке в lambda-функции!
Так на сообщения ‘Привет друг!’ или ‘Привет, друг’ Бот не будет реагировать.

Для использования Text импортируем класс фильтра в проект:

from aiogram.dispatcher.filters import Text

и применяем его в качестве параметра декоратора @dp.message_handler() для проверки входящего сообщения на полное совпадение фраз:

@dp.message_handler(Text(equals='Привет, друг!'))
async def with_puree(message: types.Message):
    await message.reply('И тебе не хварать!')

Использование декоратора @dp.message_handler() без параметров позволит создать функцию-обработчик “любых” соощений, т.е. бот будет реагировать на любые (абсолюно все) сообщения, которые не “перехватили” другие обработчики:

@dp.message_handler()
async def echo_message(msg: types.Message):
    """ Этот обработчик будет отправлять сообщение пользователя обратно """
    await bot.send_message(msg.from_user.id, msg.text)

Бот может реагировать не только на команды, но и на любые другие сообщения.

Отправка сообщений ботом.

Бот может отправлять сообщения напрямую пользователю или в группу пользователей. Это зависит от того, откуда бот получил сообщение:

  • если пользователь отправл сообщение напрямую боту, бот ответит напрямую пользователю,
  • если бот получил сообщение из группы, то и ответ он отправит в группу.

Бот может отправлять несколько типов сообщений: ответ и цитату.

Чтбы отправить ответ на соощение, воспоьзуйтесь асинхронным методом answer() объекта message:

await message.answer('Привет!')

Чтобы отправить цитату на сообщение пользователя (процетировать сообщение), воспользуйтесь асинхронным методом reply() объекта message:

await message.reply('Это цитата Вашего сообщения!')

Отправка сообщений конкретному пользователю или в кнкреный чат

Бот может направить сообщение конкретному пользователю или в конкретный чат. Для этого воспользуйтесь методом send_message() объекта bot:

@dp.message_handler(Text(equals='Какое-то сообщение'))
async def answer_to_user(message: types.Message):
    user_id = message.from_user.id  # узнаём id пользователя,
                                    # чтобы отправить ему сообщение
    await bot.send_message(user_id, 'Ответ на сообщение')

Для отправки сообщения в конкретный чат, нужно узнать id чата:

@dp.message_handler(Text(equals='Какое-то сообщение в чате'))
async def answer_to_chat(message: types.Message):
    chat_id = message.chat.id  # для отправки в конкретный чат
    await bot.send_message(chat_id, 'Ответ на сообщение для чата')

Если необходимо отправить не просто сообщение, а цитату, передайте методу send_message() именованный параметр reply_to_message_id, которому присвойте идентификатор сообщения, которое нужно процетировать:

@dp.message_handler(Text(equals='Какое-то сообщение в чате'))
async def reply_to_chat(message: types.Message):
    chat_id = message.chat.id  # id чата, в который пришло сообщение
    message_id = message.message_id  # id сообщения, которое нужно процетировать
    await bot.send_message(chat_id, 'Цитата сообщения', reply_to_message_id=message_id)

Форматирование текста сообщений

Telegram позволяет форматировать текст сообщения, делая его

  • жирным,
  • курсивным,
  • моноширинным,
  • зачеркнутым,
  • подчеркнутым
  • скрытым и
  • ссылкой.

Для разметки текста используется два типа разметки: Markdown и HTML. Разметка позволяет с помощью разметочных последовательнойстей (тегов) задать формат всему тексту или его части.

Обратите внимание на именованный аругмент parse_mode="MarkdownV2" метода answer(). Именно он говорит Telegram`у как нужно интерпретировать теги. Для HTML разметки значеине данного параметра должно быть соответствующим: parse_mode="HTML".

Пример с MarkdownV2:

@dp.message_handler(commands=['formats'])
async def cmd_show_formats(message: types.Message):
    msg = '*bold text * \n'\     # Жирный текст
          '_italic text_ \n'\    # Курсив
          '__underline__ \n'\    # Подчёркнутый текст
          '~strikethrough~ \n'\  # Зачёркнутый текст
          '||spoiler|| \n'\      # Скрытый текст
          '[inline URL](http://www.example.com/) \n'\  # Ссылка
          '[inline mention of a user](tg://user?id = 123456789) \n'\  # Ссылка на профиль пользователя
          '`inline fixed-width code`'  #  Моноширинный текст (код)
    await message.answer(msg, parse_mode="MarkdownV2")

Пример с HTML:

@dp.message_handler(commands=['formats'])
async def cmd_show_formats(message: types.Message):
    msg_html = text(
        '<b>bold</b>, <strong>bold</strong>',
        '<i>italic</i>, <em>italic</em>',
        '<u>underline</u>, <ins>underline</ins>',
        '<s>strikethrough</s>, <strike>strikethrough</strike>, <del>strikethrough</del>',
        '<b>bold <i>italic bold <s>italic bold strikethrough</s> <u>underline italic bold</u></i> bold</b>',
        '<a href="http://www.example.com/">inline URL</a>',
        '<a href="tg://user?id=123456789">inline mention of a user</a>',
        '<code>inline fixed-width code</code>',
        '<pre>pre-formatted fixed-width code block</pre>',
        '<pre><code class="language-python">pre-formatted fixed-width code block </code></pre>')
    await message.answer(msg_html, parse_mode='HTML')

Обратите внимание на функцию text(). Она позволяет склеить аргументы в единый текст. Подключть её можно так:

from aiogram.utils.markdown import text

Кроме того, функция text() поддерживает форматирование текста в формате HTML с помощью специальных функций:

  • hbold() - сделает текст жирным
  • hitalic() - сделает текст курсивом
  • hstrikethrough() - сделает текст зачеркнутым
  • а здесь можно посмотреть полный список функций HTML форматирования текста.

Подключить эти функции можно вместе с text:

from aiogram.utils.markdown import text, hbold, hitalic, hstrikethrough

Пример использования:

@dp.message_handler(commands=['formats'])
async def cmd_show_formats(message: types.Message):
    msg = text(hbold('Жирный текст'), ',',
               hitalic('далее курсив'), ',',
               hstrikethrough('и зачеркнутый'))
    await message.answer(msg, parse_mode='HTML')
  • Подробнее про форматирования текста с помощью MarkdownV2 и HTML:

Чтобы избавиться от необходимости постоянного указания именованного параметра parse_mode="" для метода answer(), можно указать способ форматирования текста глобально, т.е. для всего Бота. для этого при создании экземпляра бота передайте конструктору именованный параметр parse_mode с нужным значением:

bot = Bot(token=TOKEN, parse_mode="HTML")

Значения параметра parse_mode:

  • MarkdownV2
  • HTML

Эмоджи в сообщении 😊

Добавить эмоджи в сообщение можно несколькими способами:

  • с помощью копирования изображения эмоджи и вставки напрямую в редактор кода PyCharm: em1 = '🤪' или em2 = '🙂'
  • с момощью коротких имен: :grinning_face_with_big_eyes: (😃)

Скопированный изображения эмоджи не нужнаются в предварительной обработке, тогда как использование коротких имен требует предварительной обработки текста.
Для использования коротких имен эмоджи в тексте необходимо установить модуль emoji:

pip install emoji

Для отображения эмоджи требуемый текст или его часть необходимо пропустить через метод emojize().

import emoji
...
@dp.message_handler(commands='emo')
async def foo_handler(message: types.Message):
    # Emoji как отдельные символы
    em1 = '🤪'
    em2 = '🙂'
    # Список коротких имен emoji
    content = [':grinning_face_with_big_eyes:',
               ':relieved_face:',
               ':beaming_face_with_smiling_eyes:']
    await message.answer(text(em1, em2))
    await message.answer(emoji.emojize(text(*content)))

Полный список эмоджи: Full Emoji List, v15.0

Обратите внимание на выражение text(*content). Функуция text() позволяет склеить аргументы в единый текст, а распаковка списка *content в данном случае ‘раскидывает’ элементы этого списка в аргументы функции text(). Тем самым получаем строку текста из списка.

Кнопки и клавиатуры

Для рабты с Ботом можно использовать “родную” клавиатуру устройства (котороый вы пользуетесь всё время) и виртуальную клавиаутуру с виртуальными кнопками. Кнопки на виртуальной клавиатуре бывают:

  • обычные (цепляются к низу экрана),
  • специальные обычные (как обычные, но отправляют не текст),
  • инлайн-кнопки (цепляются к сообщению).

Обычные кнопки

Чтобы создать обычную кнопку и воспользоваться ей, нужно:

  1. сначала создать виртуальную клавиатуру (отдельно) ***,
  2. затем создать саму кнопку (отдельно),
  3. далее “присоединить/добавить” кнопку на клавиатуру.
    *** на самом деле создаётся не клавиатура а её разметка (keyboard markup - разметка клавиатуры), но мы будем называть разметку просто “клавиатура”

Классы Клавиатуры и Кнопки находятся в модуле types AIOGRAM.

Создадим handler, который в ответ на команду /simple_button будет отправлять пользователю клавиатуру с одной обычной кнопкой:

@dp.message_handler(commands='simple_button')
async def send_simple_button(message: types.Message):
    # Создаём клавиатуру
    keyboard = types.ReplyKeyboardMarkup()

    # Создаём кнопку
    button_1 = types.KeyboardButton(text='Первая кнопка')

    # Кнопку можно создать проще:
    button_2 = "Вторая кнопка"

    # Присоединяем кнопки к клавиатуре:
    keyboard.add(button_1, button_2)

    # Т.к. кнопки будут отправлены в ответ на сообщение пользователя,
    # используем клавиатуру в методе answer()
    await message.answer('Вот Ваши кнопки, Сир!',
                         reply_markup=keyboard)

Присоединить клавиаутуру к сообщению помогает именованный параметр reply_markup='' метода answer().
Отправленные кнопки занимают много места, точнее всё место, отведенное под клавиатуру. Чтобы кнопки стали меньше (отмасштабировались под размер текста, написанного на них), нужно передать в конструктор ReplyKeyboardMarkup() именованный параметр resize_keyboard=True. Попробуйте добавить его в пример выше.

...
# Создаём клавиатуру
    keyboard = types.ReplyKeyboardMarkup(resize_keyboard=True)
...

Основной смысл кнопки в том, чтобы отправить в чат напичанный на ней текст. По-сути, кнопка - это шаблон какого-то сообщения. Нажимая на кнопки из примера выше, вы увидите, что при нажатии на соответствующую кнопку в чат бдут отправлен текст, написанный на этой кнопке.

Реакция Бота на обычные кнопки

Реакция Бота на текст из кнопок это, по-сути, обработка обычных сообщений (рассматривалась в разделе “Обработка обычных сообщений”).
Напоминаем: обработать сообщения можно с помощью фильтра Text() и с помощью обычной lambda-функции:

@dp.message_handler(Text(equals='Первая кнопка'))
async def first_button(message: types.Message):
    ''' Обработчик текста первой кнопки (с помощью фильтра Text) '''
    await message.answer('Вы нажали на первую кнопку 🙂')


@dp.message_handler(lambda message: message.text == 'Вторая кнопка')
async def second_button(message: types.Message):
    ''' Обработчик текста второй кнопки (с помощью lambda-Функции) '''
    await message.answer('Вы нажали на вторую кнопку 🙃')

Код выше обрабатывает текст кнопок из предыдущего примера.

Специальные кнопки

Специальные кнопки не являются шаблонами, т.к. отправляют “не то, что на них написано”. Такие кнопки могут отправить:

  • текущую геолокацию
  • номера телефона (пользователя)
  • и ярлык для создания опроса/викторины.
@dp.message_handler(commands='special_buttons')
async def special_buttons_handler(message: types.Message):
    # Создаём клавиатуру
    keyboard = types.ReplyKeyboardMarkup(resize_keyboard=True)

    # Спецкнопка для отправки геоданных (местоположения) пользователя
    geolocation_button = types.KeyboardButton(text='Отправить локацию',
                                              request_location=True)

    # Спецкнопка для отправки номера телефона пользователя
    contact_button = types.KeyboardButton(text='Отправить контакт',
                                          request_contact=True)

    # Прежде создания кнопки, нужно создать викторину:
    poll = types.KeyboardButtonPollType(type=types.PollType.QUIZ)
    # а после и спецкнопку для отправки этой викторины
    poll_button = types.KeyboardButton(text='Создать викторину',
                                       request_poll=poll)

    # Присоединяем кнопки к клавиатуре
    keyboard.add(geolocation_button, contact_button, poll_button)

    # Отпарвляем клавиатуру со спецкнопками в ответ на команду
    await message.answer('Что сделать?', reply_markup=keyboard)

По-сути, специальные кнопки создаются так же, как и обычные кнопки, только при их создании указываются именованные параметры request_location=True, request_contact=True или request_poll=poll в зависимости от того, какую спецкнопку мы хотим создать.
Первые кнопки простые и понятные, с кнопкой опроса сложнее.
Подробнее про создание опросов на Python и AIOGRAM можно прочитать тут: Опросы v2.0

Инлайн-кнопки

В отличие от обычных кнопок, инлайн-кнопки цепляются к сообщению (отправляются прямиком в чат). Инлайн-кнопки могут быть:

  • URL (ссылки). Поддерживают только HTTP(S) или TG протоколы. Для краткости будем называть их URL-кнопками.
  • Callback (кнопки для функциq обратного вызова) или Callback-кнопки (для краткости).
  • Switch (переключатель в инлайн-режим **)
    ** Специальный режим, позволяющий пользователю отправить информацию от своего имени, но с помощью Бота.

Для создания инлайн-кнопок используюся инлайн-клавиатуры types.InlineKeyboardMarkup() и инлайн-кнопки types.InlineKeyboardButton().

URL-кнопки

@dp.message_handler(commands="inline_url")
async def cmd_inline_url(message: types.Message):
    # Создаём список инлайн-кнопок с помощью класса InlineKeyboardButton().
    # URL-кнопки кроме параметра text имеют параметр url.
    buttons = [
        types.InlineKeyboardButton(text="GitHub", url="https://github.com"),
        types.InlineKeyboardButton(text="Оф. канал Telegram", url="tg://resolve?domain=telegram")
    ]
    
    # Создаем инлайн-клавиатуру.
    # Параметр row_width отвечате за количество кнопок в строке.
    # При row_width=1 в строке будет помещаться одна кнопка.
    # Если убрать row_width, будет использоваться значение по умолчанию - 3.
    keyboard = types.InlineKeyboardMarkup(row_width=1)
    
    # Добавляем кнопки на клавиатуру.
    keyboard.add(*buttons)
    await message.answer("Кнопки-ссылки", reply_markup=keyboard)

Обратите внимание на способ создания кнопок с помощью списка (buttons = [...]).
Список избавляет от необходимости создавать отдельные переменные для каждоый кнопки. К тому же с помощью списка кнопки добавляются простой распаковкой оного внутри метода add() клавиатуры: keyboard.add(*buttons).

Callback-кнопки

Callback-кнопки имею специальный атрибут callback_data. Он позволяет определить, “что было нажато и что надо сделать”. Нажатие на Callback-кнопку позволяет сделать что угодно: от заказа пиццы до перезагрузки сервера.

Как пример, напишем генератор случайного числа (т.е. один из вариантов поведения Бота), который будет отправлять пользователю число от 1 до 10.
Для этого сначала напишем handler, который по команде random будт отправлять сообщение с Callback-кнопкой:

@dp.message_handler(commands='random')
async def make_random(message: types.Message):
    # Создаём инлайн-клавиатуру
    keyboard = types.InlineKeyboardMarkup()

    # Создаём Callback-кнопку
    # Указываем параметр callback_data (он превращает инлайн-кнопку в Callback)
    callback_button = types.InlineKeyboardButton(text='Нажми меня!',
                                                 callback_data='random_value')

    # Добавляем кнопку на клавиатуру
    keyboard.add(callback_button)

    # Отправляем клавиатуру с кнопкой в ответ на запрос
    await message.answer('Нажмите на кнопку, чтобы получить число от 1 до 10',
                         reply_markup=keyboard)

Если сейчас нажать на эту кнопку, то ничего не произойдет (Бот ничего не сделает), но, в нижнем правом углу кнопки появятся часики (иконка циферблата).
При нажатии Callback-кнопка отправляет Боту Callback-запрос (или запрос на выполнение функции обратного вызова).
Чтобы обработать Callback-запрос, используется декоратор callback_query_handler() (вместо message_handler()):

@dp.callback_query_handler(text='random_value')
async def send_random(call: types.CallbackQuery):
    """ Функция принимает параметр call (запрос) - объект типа CallbackQuery.
    В отличие от message_handler(), который работает с объектом типа Message
    (т.е. сообщение от пользователя), callback_query_handler() работает с
    объектом CallbackQuery (можно назвать его 'сообщение от кнопки') """
    
    # Обратите внимание на атрибут message объекта call - это сообщение, к
    # которому была приклеена callback-кнопка.
    await call.message.answer(str(random.randint(1, 10)))

Теперь при нажатии на callback-кнопку “Нажми меня!”, Бот отправит случаное число от 1 до 10, НО!.. только одно. При этом на кнопке снова появятся часики (иконка циферблата) в нижнем правом углу.

Зачем нужны эти часики? Дело в том, что сервер Telegram ждёт от Бота подтверждения о доставке колбэка, иначе в течение 30 секунд будет показывать эту специальную иконку и заблокирует отправку callbacл`ов на это время.

Информировать сервер Telegram о доставке callback`а можно с помощью вызова метода answer() у объекта CallbackQuery (параметр call в сигнатуре функции send_random(), cм. код выше.).

Доработаем функцию send_random():

@dp.callback_query_handler(text='random_value')
async def send_random(call: types.CallbackQuery):
    await call.message.answer(str(random.randint(1, 10)))
    
    # Отправляем серверу Tg уведомление о том, что callback запрос доствлен
    await call.answer()

Вызов метода answer() с параметрами text='' покажет пользователю всплывающее уведомление (содержимое text) после выполнения callback-функции, а параметр show_alert=True позволит отобразить это сообщение в модальном окне (с кнопкой ‘ok’):

@dp.callback_query_handler(text='random_value')
async def send_random(call: types.CallbackQuery):
    await call.message.answer(str(random.randint(1, 10)))
    
    # Вариант уведомления с отображением модального окна
    await call.answer(text='Спасибо, что воспользовались Ботом.', show_alert=True)

Многоразовые и одноразовые клавиатуры

Обычная клавиатура ReplyKeyboardMarkup() с кнопками никуда не исчзает и “висит” внизу чата постоянно - она многоразовая и позволяют пользоваться ей длительное время.
Чтобы сделать одноразовую клавиатуру (клавиатуру “под одну задачу”), нужно у ReplyKeyboardMarkup() указать именованный параметр one_time_keyboard=True, тогда клавиатура будт отправляться на “один раз”.

keyboard = types.ReplyKeyboardMarkup(one_time_keyboard=True,
                                     resize_keyboard=True)

Чтобы удалить многоразовую клавиатуру, которая была отправлена ранее, нужно отправить сообщение со специальной “удаляющей” клавиатуройтипа ReplyKeyboardRemove():

@dp.message_handler(commands='delkb')
async def del_kb(message: types.Message):
    await message.reply('Удаляем...', reply_markup=types.ReplyKeyboardRemove())

Работа с файлами

Файлы часто используются польователями и Ботами для различных целей, например, пользователи отправляют Боту фотографии, чтобы тот применил к ним фильтры (как-то обработал) и отправил обработанный файл обратно. Бот может работать с разными типами файлов:

  • фотографии
  • изображения
  • видео
  • аудио
  • голос
  • документы

Отправка файлов пользователю

Отправка изображений

Чтобы отправить изображение пользователю, оно должно находиться на сервере, где располагается Бот.Для отправки файлов будм использовать объект bot (экземпляр классаBot):

@dp.message_handler(commands='kitty')
async def send_kitty(message: types.Message):
    """ Данная функция отправит изображение кошки по команде """
    
    # Узнаём id чата, куда хотим отправить изображение
    # В данном случае чат между конкретным пользователем и Ботом можно
    # определить с помощью id пользователя
    user_id = message.from_user.id

    # Читаем файл в переменную в бинарном режиме 'rb'
    photo = open('media/photos/seraya-koshka-s-zhltymi-glazami.jpg', 'rb')
    
    # с помощью метода send_photo() отправляем файл пользователю
    await bot.send_photo(user_id, photo)

Отправка голосового сообщения

Голосовое сообщение - это поток аудиоданных, который можно напрямую прослушать в чате. Родной формат фалов Telegram *.oga, однако в качестве голосового сообщения можно отправить и файл в формате *.mp3:

@dp.message_handler(commands='voice')
async def send_voice(message: types.Message):
    # Получаем id пользователя
    user_id = message.from_user.id

    # Читаем файл в переменную в бинарном режиме 'rb'
    voice = open('media/voice/file_10.oga', 'rb')

    # с помощью метода send_voice() отправляем файл пользователю
    await bot.send_voice(user_id, voice)

Отправка аудио

Отправка аудиофайла похожа на отправку голосового сообщения, но в чате отображаться такой файл будет по-другому. Для отправки используется метод send_audio():

@dp.message_handler(commands='mp3')
async def send_mp3(message: types.Message):
    # Получаем id пользователя
    user_id = message.from_user.id

    # Читаем файл в переменную в бинарном режиме 'rb'
    mp3 = open('media/mp3/zvuk_fakela.mp3', 'rb')
    
    # С помощью метода send_audio() отправляем файл
    await bot.send_audio(user_id, mp3)

Отправка видео

В качетве примера видеофайла в данном случае использован файл в формате “*.mp4”:

@dp.message_handler(commands='video')
async def send_video(message: types.Message):
    # Получаем id пользователя
    user_id = message.from_user.id

    # Читаем файл в переменную в бинарном режиме 'rb'
    video = open('media/video/sample-5s.mp4', 'rb')

    # Отправляем файл с помощью send_video()
    await bot.send_video(user_id, video)

Отправка медиа-группы

Медиа-группа - это группа из нескольких файлов, которая высылается одним сообщением. Эта группа может быть подписана.
Чтобы создать медиа-группу необходимо исползовать объекты группы InputMedia (находятся в types), соответствующие формату файлов, которые нужно поместить в Медиа-группу:

  • types.InputMediaPhoto - для фотографий/изображений
  • types.InputMediaAudio - для аудио-файлов
  • types.InputMediaVideo - для видео
  • types.InputMediaDocument - документов
  • types.InputMediaAnimation - GIF-анимаций

Добавлять все подряд типы файлов в Медиа-группу нельзя. Типы файлов, которые можно отправить в одной Медиа-группе:

  • InputMediaPhoto + InputMediaVideo + types.InputMediaAnimation

Типы types.InputMediaAudio и types.InputMediaDocument не могут быть смешаны с другими медиа типами.

Подробнее про медатипы можно прочитать:

  • Документация AIOGRAM: InputMedia
  • Документация Telegram: InputMedia

Создадим Медиа-группу изображений и отправим её в чат:

@dp.message_handler(commands='mediagroup')
async def send_media_group(message: types.Message):
    # id чата, из которого пришла команда /mediagroup
    chat_id = message.chat.id

    # Для представления медиагруппы будм использовать список
    media_group = []

    # Открываем файлы изображений для медиа-группы
    image1 = open('media/pictures/car_1.jpg', 'rb')
    image2 = open('media/pictures/ferrari.jpg', 'rb')
    image3 = open('media/photos/seraya-koshka-s-zhltymi-glazami.jpg', 'rb')

    # Добавляем в медиагруппу объекты InputMediaPhoto, созданные на основе
    # открытых файлов
    media_group.append(types.InputMediaPhoto(image1))
    media_group.append(types.InputMediaPhoto(image2))
    media_group.append(types.InputMediaPhoto(image3))

    # Отправляем Медиа-группу в чат (или пользователю) по id
    await bot.send_media_group(chat_id, media_group)

Медиагруппы можно создавать без промежуточных переменных. Упростим пример выше:

@dp.message_handler(commands='mediagroup')
async def send_media_group(message: types.Message):
    # id чата, из которого пришла команда /mediagroup
    chat_id = message.chat.id

    # Добавляем объекты InputMediaPhoto напрямую в список и открываем файлы
    # "внутри" этих объектов:
    media_group = [
        types.InputMediaPhoto(open('media/pictures/car_1.jpg', 'rb')),
        types.InputMediaPhoto(open('media/pictures/ferrari.jpg', 'rb')),
        types.InputMediaPhoto(open('media/photos/seraya-koshka-s-zhltymi-glazami.jpg', 'rb'))
    ]

    # Отправляем Медиа-группу в чат (или пользователю) по id
    await bot.send_media_group(chat_id, media_group)

Для Медиа-групп можно делать текстовое описание, которое будет отображено под ней. Для этого используйте именованный параметр caption='' только у первого объекта Медиа-группы:

...
# Добавляем в медиагруппу объекты InputMediaPhoto, созданные на основе
    # открытых файлов
    media_group.append(types.InputMediaPhoto(image1,
                                             caption='Описание медиа группы'))
    media_group.append(types.InputMediaPhoto(image2))
    media_group.append(types.InputMediaPhoto(image3))
...

Если добавить caption='' к нескольким объектам, описание группы не будт отображено. В этом случае каждый caption='' можно будет увидеть на отдельном изображении при его открытии.

Медиа группа с разными типами документов:

...
media_group = [
        types.InputMediaDocument(open('documents/example.pdf', 'rb')),
        types.InputMediaDocument(open('documents/example.docx', 'rb'))
    ]
...

Сокрытие токена в переменной окружения.

  1. Скопируйте ваш токен в буфер обмена (Ctrl + C).
  2. Пройдите в меню Run | Edit Configurations. Откроется окно Run/Debug Configurations.
  3. В группе настроек Enviroment найдите поле Enviroment Variables (обычно там отображается PYTHONUNBUFFERED=1).
  4. Нажмите на иконку списка в правой части этого поля. Откроется окно Enviroment Variables.
  5. Нажмите на иконку “плюс”. Это активирует столбец Name и позволит ввести имя переменной окружения. Введите TOKEN.
  6. Слелайте двойной клик в столбце Value напротив TOKEN и вставьте скопированный ранее токен, нажав Ctrl + V.
  7. Нажмите ОK дважды (для окон Enviroment Variables и Run/Debug Configurations).
  8. Подключите модуль os для работы с переменными окружения и запишите токен из переменной окружения в константу:
import os
TOKEN = os.environ['TOKEN']
